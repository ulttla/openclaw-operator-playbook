# Contributing

Contributions are welcome if they improve the public operator playbook without exposing private infrastructure.

## Good contributions

- Clearer templates.
- Safer redaction patterns.
- Better routing criteria.
- Reusable examples.
- Corrections that align examples with current OpenClaw behavior.

## Please avoid

- Private config dumps.
- Provider keys or account IDs.
- Vendor-specific secrets.
- Claims without evidence.
- Fictional config keys presented as real OpenClaw config.

## Pull request checklist

- [ ] No secrets, tokens, private IDs, or private hostnames.
- [ ] New examples are public-safe and use placeholders.
- [ ] If a file is conceptual, it says so clearly.
- [ ] If a file describes OpenClaw behavior, it matches official docs or says it is an operator pattern.
- [ ] Markdown links were checked.

## Formatting

- Use short sections and practical examples.
- Prefer tables for decision rules.
- Keep templates copyable.
- Avoid private personal naming unless it is intentionally public context.
