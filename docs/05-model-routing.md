# Model Routing

Model routing should be based on task requirements, not hype.

## Criteria

- Context length.
- Code editing strength.
- Latency.
- Cost.
- Privacy needs.
- Tool integration quality.
- Failure mode tolerance.

## Example routing policy

Use fast and inexpensive models for low-risk summarization. Use stronger coding agents for repository edits. Use local models for sensitive review when quality is acceptable. Use explicit approval for costly or external actions.
