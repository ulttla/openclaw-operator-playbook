# Glossary

| Term | Meaning |
|---|---|
| Approval gate | A boundary where the operator must explicitly approve action before proceeding |
| Check-in | A short progress update during a long work session |
| Checkpoint | A compact state record that makes work resumable after reset or interruption |
| Closeout | The end-of-session evidence summary: what changed, how it was checked, what remains |
| Evidence | A test, diff, build, screenshot, link check, review note, or named blocker |
| Harness | A routing system that sends work to the right model, tool, or reviewer lane |
| Lane | A specialized execution or review path, such as builder, reviewer, research, risk, or local smoke |
| Long Work Window | A bounded supervised work session for substantial AI-assisted work |
| Operator | The human or lead assistant responsible for scope, risk, approval, and final judgment |
| Operator knowledge base | A structured personal wiki or notes system used by the AI workstation |
| Raw notes | Append-only capture before cleanup or promotion |
| Redaction | Removing private information before publishing artifacts |
| Reviewer packet | A compact handoff containing scope, artifacts, risks, evidence, and questions |
| Staging | The cleanup layer between raw notes and durable wiki pages |
| Workstation | The combined local tools, models, skills, notes, and policies used for AI work |
