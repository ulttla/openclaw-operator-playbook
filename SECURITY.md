# Security Policy

This repository is public-safe by design. Please do not open issues containing secrets, tokens, private logs, account IDs, credentials, channel IDs, or private hostnames.

## Reporting a sensitive finding

If you notice accidentally exposed sensitive information, report it privately to the repository owner instead of opening a public issue.

If GitHub Security Advisories are enabled for the repository, use a private advisory. Otherwise, contact the owner through their public profile contact path and include only the minimum necessary details.

## What to avoid in reports

Do not paste:

- full tokens or keys;
- OAuth files;
- complete private logs;
- private infrastructure maps;
- real backup destinations.

Use redacted snippets and file paths instead.
